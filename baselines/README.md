# Baseline numbers

This directory contains the latest reference baseline numbers for the models
shipped with OpenLithoHub. Files are regenerated by

```
python scripts/generate_baselines.py --synthetic --limit 8 --output baselines/
```

## Files

- `results.json` — structured per-model metrics. Drives leaderboard
  submissions and downstream comparisons.
- `results.md` — human-readable markdown table mirrored into
  `docs/benchmarks.md`.

## Reproducing

The default invocation runs against eight hand-rolled 64×64 synthetic
patterns (square, line, line/space, T, L, cross, contacts, dense lines).
This avoids any external dataset download and is what the published numbers
in the docs reflect.

To run against a real LithoBench checkout:

```
python scripts/generate_baselines.py \
  --data-root /path/to/lithobench --limit 16 --output baselines/lithobench/
```

Pass `--models dummy-identity levelset-ilt neural-ilt` to evaluate any
subset of the registered baselines. Models that fail to register (e.g.
`neural-ilt` when its optional weights are missing) are skipped gracefully.

## Notes on the current numbers

- `dummy-identity` is the no-op baseline — it copies the design straight
  through. EPE is by construction zero on synthetic patterns where
  design == target. It exists to verify that the metric pipeline itself is
  not broken.
- `levelset-ilt` runs 200 iterations of gradient-descent ILT against the
  default Gaussian PSF forward model. Numbers reflect that default; pass
  `forward_model="hopkins"` in user code to try the partial-coherent
  Hopkins model added in the same release.
- `neural-ilt` uses an untrained U-Net by default. The numbers shown are
  intentionally the "no-weights" baseline; they will improve substantially
  once `from_pretrained()` weights are available.
