# Baseline numbers

This directory contains the latest reference baseline numbers for the models
shipped with OpenLithoHub. Files are regenerated by

```
python scripts/generate_baselines.py --synthetic --limit 8 --output baselines/
```

## Files

- `results.json` — structured per-model metrics. Drives leaderboard
  submissions and downstream comparisons.
- `results.md` — human-readable markdown table mirrored into
  `docs/benchmarks.md`.

## Reproducing

The default invocation runs against eight hand-rolled 64×64 synthetic
patterns (square, line, line/space, T, L, cross, contacts, dense lines).
This avoids any external dataset download and is what the published numbers
in the docs reflect.

To run against a real LithoBench checkout:

```
python scripts/generate_baselines.py \
  --data-root /path/to/lithobench --limit 16 --output baselines/lithobench/
```

Pass `--models dummy-identity rule-based-opc levelset-ilt openilt neural-ilt` to
evaluate any subset of the registered baselines. Models that fail to
register (e.g. `neural-ilt` when its optional weights are missing) are
skipped gracefully.

## Notes on the current numbers

- `dummy-identity` is the no-op baseline — it copies the design straight
  through. EPE is by construction zero on synthetic patterns where
  design == target. It exists to verify that the metric pipeline itself is
  not broken.
- `rule-based-opc` is the geometric reference: morphological edge bias by
  one pixel plus a small line-end extra-bias. No optical simulation, no
  learning — this is what plain dilation buys you, and what every learned
  model has to beat.
- `levelset-ilt` runs 200 iterations of gradient-descent ILT against the
  default Gaussian PSF forward model. Numbers reflect that default; pass
  `forward_model="hopkins"` in user code to try the partial-coherent
  Hopkins model added in the same release.
- `openilt` is the OpenILT-style SimpleILT baseline (clean-room PyTorch
  reimplementation, MIT-licensed upstream). Optimises the MOSAIC
  L2 + PVBand objective with SGD across a 3-corner dose/defocus sweep,
  vs. `levelset-ilt`'s single-corner fidelity. The higher PV-band on
  the synthetic-8 set reflects the broader corner span — `levelset-ilt`
  optimises only at nominal, so it lands a tighter nominal mask but
  isn't directly comparable on PV-band.
- `neural-ilt` is the U-Net mask predictor. Public v0.1 seed weights
  are available at [`openlithohub/neural-ilt-v0.1`](https://huggingface.co/openlithohub/neural-ilt-v0.1)
  on HuggingFace; `NeuralILTModel(pretrained=True)` and
  `scripts/generate_baselines.py --pretrained` (default) load them.
  v0.1 was trained on synthetic dummy layouts, so the numbers above
  reflect identity-with-rounding behaviour rather than real OPC
  corrections — a v1.0 LithoBench-trained release is planned. With
  `--no-pretrained` the model falls back to a randomly-initialised
  U-Net.
